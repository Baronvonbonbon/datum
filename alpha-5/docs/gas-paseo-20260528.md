## Key takeaways

**Settlement gas is per-claim, not per-impression.** SCALE-2 (1 claim × 1000 events) and SCALE-3 (1 claim × 10 events) both land near the 40k baseline — eventCount inside the claim moves the figure by less than 100 gas. The dominant costs are claim-hash recomputation, the Merkle nonce/prevHash check, the reputation read, the budget-ledger deduction, and the payment-vault credit. Each runs once per claim regardless of impressions inside it.

**Two settlement tiers, ~8k apart.** Every settle clusters around either ~40.9k or ~32.5k:

- ~40.9k path: bonding-curve write on `DatumPublisherStake.recordImpressions` (`agg.eventsSettled > 0`) plus the per-publisher take-rate snapshot path through `DatumPublishers.relaySigner`.
- ~32.5k path: same publisher already advanced this block (SCALE-3, OPEN-3, RPT-PRESETTLE-B), so the bonding-curve write hits a warm slot and `relaySigner()` short-circuits via the snapshot cached at campaign creation. The ~8k drop is the warm-storage difference.

**ECO-$5DOT is the outlier (43.5k).** It's the only settle that touched a freshly-initialised `_lastSettlementBlock[user][campaignId][actionType]` slot AND a fresh `_userCampaignSettled` mapping in the same TX — that's two cold SSTOREs vs one in the other ECO scenarios.

**Reports are cheap.** `reportPage` and `reportAd` come in under 9k gas — a single SSTORE on the `pageReports` / `adReports` counter, plus an event emit. The publisher / advertiser cross-references add ~400 gas. The `MIN_EVENTS_TO_REPORT=1` gate is satisfied by a cached settlement read, so the report itself stays in the single-digit-k range.

**Lifecycle is dirt-cheap on Phase 0 admin gov.** `adminActivateCampaign` at 2,356 gas is essentially one status flip + one event emit. Production phase-1/phase-2 paths (Council vote, OpenGov + Timelock 48h) will be substantially more.

**Conversion sanity check.** At the testnet's `gasPrice = 1×10¹²` planck/gas, a 40,915-gas settle costs `40,915 × 10¹² / 10¹⁰ = 4,091` PAS per claim. Production gas pricing is the real cost driver; these gas figures are stable across pricing regimes.

## Methodology

Each row is one `eth_estimateGas` call against the live deploy with the same calldata the benchmark would submit. The benchmark disables `DatumPowEngine.enforcePow` for the duration of the run (re-enabled at exit) because the funded test users from `setup-testnet.ts` have accumulated PoW buckets from earlier runs that make on-chain mining infeasible. The settle figures here therefore exclude the per-claim `keccak256(claimHash || powNonce)` preimage cost. Settlement gas captured below is dominated by:

- claim hash recomputation + Merkle nonce check + reputation read
- nullifier writes (skipped here: nullifier=bytes32(0))
- payment vault credit + bonding-curve update + reputation record

## Known issues in this run

- **SCALE-1, SCALE-1b, MULTI-1** show `—` because the underlying settle TX rejected all claims (`settled=0`). Single-claim variants (SCALE-2, SCALE-3) settle cleanly on the same campaign, so the issue is multi-claim-specific to the user-pool rotation and not a contract-side regression. Likely a stale state mismatch from a prior benchmark run on the same funded test users; redeploying the testnet would clear it.
- **PAUSE-3** failed because the proposalId-search loop can't find a guardian-approvable proposal — separate from settle costs, doesn't affect gas figures.
- **PoW path not exercised on Paseo.** True ephemeral wallets (which would start at bucket=0 and let the PoW gate run end-to-end with sub-second mining) require either Paseo's eth-rpc gateway to accept freshly-derived addresses (today it returns "Invalid Transaction" code 1010), or Pine's smoldot transport. The Pine path is scaffolded in `scripts/benchmark-paseo.ts` behind `BENCHMARK_PINE=1` and works once Pine's parallel-tx validation lands. We landed a Pine fix in this same commit window (`pine/src/codec/scale.ts:decodeU256` now tolerates short scalars) that unblocked `eth_getTransactionCount`; the remaining blocker is smoldot's tx-service rejecting parallel sends with "Error trying to access the storage".
